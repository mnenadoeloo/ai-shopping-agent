# AI Shopping Agent

AI Shopping Agent — это агентная система на базе LLM, которая помогает пользователю подобрать товары по текстовому описанию, объясняет рекомендации и предлагает альтернативы с учётом бюджета, предпочтений и доступности товаров.

Система объединяет несколько компонентов: retrieval по каталогу товаров, ранжирование результатов, генерацию ответов через LLM и интеграции с внешними API (цена, наличие, акции). Агент поддерживает диалоговый сценарий и сохраняет контекст взаимодействия с пользователем.

Цель проекта — продемонстрировать архитектуру агентной LLM-системы для задач e-commerce рекомендаций.

Проект реализован как Proof-of-Concept для учебных целей.

---

# Product Logic

Пользователь может обратиться к агенту с запросами:

- "Подбери ноутбук для программирования до 1500$"
- "Что подарить другу, который любит настольные игры?"
- "Найди похожие товары дешевле"
- "Есть ли альтернативы этому товару?"

Агент выполняет несколько шагов:

1. анализирует намерение пользователя
2. ищет релевантные товары в каталоге
3. ранжирует найденные результаты
4. проверяет цену и наличие
5. формирует объяснение рекомендаций

---

# Dataset

Для Proof-of-Concept используется подмножество открытого e-commerce датасета (~10k товаров), содержащего:

- название товара
- описание
- категорию
- цену
- рейтинг

Дополнительно были добавлены синтетические поля для имитации реального каталога:

- stock (наличие)
- popularity
- discount

Данные индексируются в векторной базе для semantic search.

# System Architecture

Система состоит из нескольких модулей.

## User Interface

Чат-интерфейс для взаимодействия пользователя с агентом.

## LLM Orchestrator

Основной компонент, который управляет логикой агента:

- анализирует намерение пользователя
- выбирает необходимые инструменты
- формирует итоговый ответ

## Product Retrieval

Поиск товаров в каталоге с использованием:

- dense embeddings
- vector search

Retrieval возвращает top-k кандидатов.

## Reranking

Результаты поиска ранжируются с помощью cross-encoder модели для повышения релевантности рекомендаций.

## External APIs

Интеграции:

- price API
- availability API
- promotion API

Это позволяет учитывать реальные ограничения.

## Memory

Система хранит:

- историю диалога
- предпочтения пользователя
- предыдущие рекомендации

Это позволяет улучшать персонализацию.

# Edge Cases

Система должна корректно обрабатывать:

- слишком общий пользовательский запрос
- бюджет ниже стоимости товаров
- отсутствие товара в наличии
- противоречивые требования
- неоднозначные запросы

В таких случаях агент может:

- задавать уточняющие вопросы
- предлагать альтернативные товары
- объяснять ограничения.
